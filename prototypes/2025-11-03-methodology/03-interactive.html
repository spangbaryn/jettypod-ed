How Project States and Feature Modes Interact

Internal Project

  • All features: Speed ‚Üí Stable
  • Features live in Stable Mode
  • No Production Mode (project isn't External)
  • Perfect for MVPs and internal tools
  • No security overhead

External Project

  • Customer features: Speed ‚Üí Stable ‚Üí Production
  • Internal tools: Speed ‚Üí Stable (stay here)
  • Infrastructure provides security baseline
  • Production Mode for customers only
  • Admin dashboards stay in Stable

Making the Transition

Going from Internal to External is itself a set of features

Infrastructure Features to Build

  • Security Hardening (SSL/TLS, rate limiting, DDoS protection)
  • Monitoring Setup (24/7 monitoring, alerting, logging)
  • Compliance Certification (GDPR, HIPAA, SOC2)
  • Feature-specific Production hardening (CSRF, session security, SQL injection protection)

Once ANY external users can access your project, it must be External
